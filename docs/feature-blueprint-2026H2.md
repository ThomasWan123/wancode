# WanCode 新功能蓝图（2026 H2 草案）

> 状态：**冻结（参考文档）**。经外部评审与项目决策（2026-07-22），
> 下一版优先级不由本蓝图决定，由 10–14 天真实项目 dogfooding 的失败
> 频率数据决定（记录于 docs/dogfood-log.md）。本文仅作方向候选池。
> 性质：研究与决策文档，不是承诺清单。修订：2026-07-22。
> 输入：当前能力盘点（v0.18.0）、对标 Claude Code / Codex 的能力差距、
> 多模型定位的差异化机会。每个方向给出 价值/成本/依赖 与一句话判断。

## 0. 当前位置（一句话）

核心循环（会话/工具/审批/回滚）、工作台（Diff/文件/审查/预览）、交付闭环
（审查→一键修→PR→CI 状态）、三层质量安全网都已就位；**产品短板不在
功能数量，在"多模型"这个卖点还只是接入层面的，没有产品化**。

## 1. 候选方向（按推荐度排序）

### ★★★ D1 模型路由与成本看板（差异化主打）
把"多模型"从配置项变成能力：
- **分工路由**：辅助任务（标题/摘要/recap/审查）自动走便宜快模型
  （glm-4-flash/deepseek-chat），主编码走旗舰（glm-5.2/reasoner）——
  引擎 config 已有辅助模型位，客户端做成可视化策略而不是手改 TOML。
- **成本看板**：按会话/按天统计 token 用量 × 各家牌价 = 花了多少钱。
  国内用户对"包月 Coding Plan vs 按量"极其敏感，这是真实决策痛点。
- **一键 A/B**：同一 prompt 两个模型并行跑（复用 worktree 隔离），
  diff 对比结果。评测型用户的传播点。
- 价值：高（唯一卖点的产品化）｜成本：中（2-3 版）｜依赖：引擎 usage
  数据已回传（ctx 条就是它）。**建议下版就做成本看板。**

### ★★★ D2 DeepSeek R1 / 思维链深度适配
- reasoning_content 已折叠展示，但 R1 的函数调用兼容性、思考预算控制、
  思考过程搜索/引用都没做。国产双雄的另一半吃透。
- 价值：高（用户基数）｜成本：低-中｜依赖：R1 API 行为摸底。

### ★★ D3 记忆系统产品化
- 引擎有 memory_flush/rewrite 通道但 UI 只有暗门。做：项目记忆面板
  （AGENTS.md 可视化编辑 + AI 提炼"这次学到了什么"一键入档）、
  跨会话偏好记忆。对标 Claude Code 的 auto-memory。
- 价值：中高（留存）｜成本：中｜依赖：无。

### ★★ D4 后台任务真并行（worktree 编排）
- 现有 worktree 是手动开。做：任务队列视图里"派单"——一句话任务 →
  自动开 worktree 子会话跑 → 完成后带审查报告回来 → 一键合并。
  相当于把 Review 后台会话模式推广成通用后台 agent。
- 价值：高（对标 Codex 云端 agent 的本地版）｜成本：高（3-4 版）｜
  依赖：background_sessions 机制已验证，是最好的地基。

### ★★ D5 Ollama / 本地模型一等公民
- OpenAI 兼容层理论可用但从未测过。做：预设卡 + 本地模型发现
  （ollama list）+ 上下文长度自动探测 + "敏感项目纯本地"模式徽标。
- 价值：中（隐私敏感人群）｜成本：低｜依赖：一台跑得动的机器实测。

### ★ D6 团队/分享面
- 会话导出 markdown/分享链接、团队配置模板（统一 MCP/hooks 下发）。
- 价值：中（但用户基数未到）｜成本：中｜判断：**先不做**，个人版
  打磨期分享功能是伪需求。

### ★★ D8 WanCode Work：开发者文档工作台（2026-07-23 外部评审补录）
- 参考 Claude 的 Chat/Code/Cowork 三分层。切入点**不是**通用办公助手,
  而是"开发者文档工作台"：读懂 PDF/DOCX/XLSX/PPTX 并把文档内容可靠转化
  为代码任务、测试、报告（详细四阶段路线与安全边界见外部评审原文,
  存 dogfood 日志 07-23 备注）。
- **验证前置**（dogfooding 中零代码执行）：三个黄金场景——①PRD+Excel
  需求表→实施计划→改代码；②项目代码+测试结果→测试报告；③项目状态+
  Git 历史→周报。当前仅 markdown/txt 可读,PDF/DOCX 的失败/将就频率
  即立项依据。
- 一期若立项只做"文档理解 MVP"（解析+引用来源+分块检索）,不做生成、
  不做语义 Diff、不做连接器;安全底线：文档内容永远是数据不是指令、
  永不覆盖原件、宏文件不执行。
- 价值：高（第二曲线）｜成本：高（3+ 版）｜依赖：dogfood 数据 ≥ 中频。
- 相关搭车项：链接统一路由（SafeLink：localhost 询问预览/文档进工作台/
  file:// 禁导航/mailto 系统客户端）,v0.18.2 全局拦截的长期形态。

### ★ D7 插件/MCP 市场入口
- 引擎有 marketplace 通道（附 B 审计过部分是 xAI 门控）。做精选列表
  （手工维护 10 个可用 MCP 一键装）比做"市场"务实。
- 价值：中｜成本：低（精选版）｜判断：作为 D1-D2 的搭车项。

## 3. 明确不做（本蓝图周期）

云端执行、企业权限、通用插件市场、分享链接、移动端、
自训模型。理由同 improvement-plan §3：用户基数与定位未到。
